## Summary

Phase 5.3 builds the Invoice Generation Pipeline on top of the Phase 5.2 InvoiceReadiness
and Phase 4.2 InvoiceLineItem infrastructure.

The pipeline follows Ledger doctrine strictly:

```
Approved Operational Activity
  → Financial Mutation Engine
  → Normalized Financial Records
  → InvoiceLineItems
  → InvoiceDraft           ← Phase 5.3
  → Invoice Workflow       ← Phase 5.3
```

No Job → Invoice shortcut was created.

---

## Architecture Principle

Single Source of Financial Truth preserved.

All Phase 5.3 features derive data from:

```
mockInvoiceLineItems[]
  └─> generated by the Phase 4.2 Financial Mutation Engine on Review Center approval
  └─> getJobInvoiceReadiness() aggregates them per job
  └─> generateInvoiceDraft() builds InvoiceDraft from the aggregated lines
  └─> mockInvoiceDrafts[] stores the resulting drafts
  └─> useStore() exposes invoiceDrafts, addInvoiceDraft(), updateInvoiceDraftStatus()
```

Phase 5.2 profitabilityEngine.ts was NOT modified.
getJobFinancialSummary() was NOT modified.
mockTimesheets / mockExpenses / mockInventoryMutations / mockEquipmentUsage were NOT modified.

## Status Transition Rules (enforced by invoiceBuilder.ts)

```
draft  → ready   ✓
ready  → sent    ✓
sent   → paid    ✓

draft  → sent    ✗  (must go via ready)
draft  → paid    ✗
ready  → paid    ✗  (must go via sent)
Any backward transition ✗
```
